# 第二轮审稿：证据设计与可证伪性

## 总评

**建议分数：6/10（弱接收边缘，仍需大修证据设计）**

第一轮已经解决最严重的图文实现错位。新版本把方法准确描述为“小波可靠性调制语义证据，构造逐图视觉原型，再校准文本原型”，并明确了目标类别零样本与测试时无反传的边界。剩余问题主要集中在实验设计是否能排除替代解释。

## 已解决问题

1. 标题、摘要、引言和方法已与实现一致。
2. 完整给出语义概率、小波可靠性、证据权重、视觉原型和文本原型更新公式。
3. 不再声称整个系统未使用辅助异常数据。
4. 预期数字已明确标为目标占位。

## 主要问题

### MAJOR-1：漏检子集指标仍可能产生循环定义

当前以 SemanticProto 的低分异常像素定义漏检子集，再比较 Ours 的召回。该指标天然使 SemanticProto 的召回接近零，也容易因阈值选择产生夸大。

**修复：**
- 在验证集上固定阈值，不用测试标签选阈值；
- 报告漏检子集规模；
- 同时报告 Ours 在该子集上的召回增量和在非漏检子集上的误报增量；
- 将其明确定位为诊断指标，而非主指标。

### MAJOR-2：纹理/结构类别分组可能被质疑为事后挑选

如果根据结果把提升类别归为“纹理类”，机制分析会产生后验偏差。

**修复：**
- 在看结果前冻结类别分组；
- 用可复现规则描述分组，例如数据集官方 texture/object 类别或基于类别属性的预注册列表；
- 除组均值外，报告全部类别和 bootstrap 置信区间；
- 若组间差异不显著，不声称增益集中于纹理类。

### MAJOR-3：核心方法和工程增强口径仍未完全分离

当前主结果骨架可能混入 multi-crop 和 pixel-to-image，而方法声称这些增强正交。若消融各行的后处理不完全相同，结果不能归因于原型校准。

**修复：**
- 设立“Pure”表：所有核心机制消融只使用相同基础路径；
- 设立“Enhanced”表：统一叠加 multi-crop 和 pixel-to-image；
- 核心 claim 只由 Pure 表支持，Enhanced 仅说明兼容性。

### MAJOR-4：缺少与已有频率方法的机制区分实验

相关工作强调“可靠性调制而非频率特征融合”，但实验只有 DirectFusion 一个很差的负控，可能被认为不公平。

**修复：**
- 增加 FeatureAug 基线：把 $W$ 作为额外特征或 logit bias，但不做原型构造；
- DirectFusion、FeatureAug、SemanticProto、Ours 使用同一输入和后处理；
- 若 FeatureAug 也有效，论文应将贡献收敛为“逐图原型是更稳定的使用方式”，而非唯一正确方式。

### MINOR-1：摘要仍略偏流程描述

可以压缩模块细节，把中心矛盾写得更锋利：固定语义原型缺实例参照；频率可衡量局部证据可靠性但不能独立判别。

### MINOR-2：占位图位置和正文阅读顺序

定性占位图当前浮到附录后，应在最终排版中放回分析段附近，或暂时移动到附录并在正文引用。

## 第二轮修改目标

1. 重构实验为 Pure / Enhanced 两条口径。
2. 加入 FeatureAug 公平频率基线。
3. 严格定义漏检子集指标和类别分组协议。
4. 压缩摘要和贡献，强化中心矛盾。
5. 把所有“如果结果符合目标则说明”写成明确判定规则。
